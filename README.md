REST-сервис для управления заявками на выплату средств. Каждая заявка создаётся через API и обрабатывается асинхронно с использованием Celery.
Проект имеет модель данных, которая реализует сущность «Заявка на выплату» с таким набором полей, как:
идентификатор;
сумма выплаты;
валюта;
реквизиты получателя;
статус заявки;
дата создания;
дата обновления.

Реализованы следующие эндпоинты:
GET /api/payouts/ — список заявок
GET /api/payouts/{id}/ — получение заявки по идентификатору
POST /api/payouts/ — создание новой заявки
PATCH /api/payouts/{id}/ — частичное обновление заявки (прежде всего — статуса)
DELETE /api/payouts/{id}/ — удаление заявки

При создании заявки запускается Celery-задача, которая принимает идентификатор созданной заявки;
имитирует обработку заявки (задержка, логирование, простая проверка);
изменяет статус заявки после обработки.
Результат выполнения кода и демонстрация его записи в БД, происходит по эндпоинту "http://127.0.0.1:8000/api/payouts/", результат демонстрируется в формате Json.
Проект подразумевает наличие SQLite-БД, которая отлично подходит для функционала данного проекта, наличие PostgreSQL-ДБ не является целесообразным.

Запуск проекта: 
установка всех зависимоствей (pip install -r requirements.txt),
наличие у пользователся POSTMAN, для осуществления запросов по "http://127.0.0.1:8000/api/payouts/"
наличие у пользователся REDIS, для возможности выполнения задач Celery
Celery Worker запускается командой для асинхронного выполнения задач: celery -A payout_project worker --loglevel=info -P gevent

Тестирование:
Проект имеет 100% покрытие тестами. Функционал обрабатывается как unit, так и интеграционными тестами.
запуск тестирования: python manage.py test
